Pesquisa com autocomplete

Problema

Um dos problemas é a complexidade no momento da pesquisa, bem como a visualização dentre as várias opções disponíveis. Para diminuir a complexidade da consulta ela pode ser dada por vários atributos e não apenas um.

Solução

Resolvendo o problema da visualização, foi usado o autocomplete. Que faz a busca em todas as Hierarquias por parte da palavra já digitada.

Digite uma parte da palavra ou frase desejada, visualize o autocomplete gerado e selecione sua opção.

Quando Usar

  • Use quando as sugestões de preenchimento automático podem ser retiradas de um conjunto de dados gerenciável em tamanho.
  • Use quando a velocidade de entrada é um objetivo importante.
  • Use quando a precisão da entrada é um objetivo importante.
  • Use quando o número de itens seria muito grande ou inconveniente para exibir em uma caixa suspensa padrão.
  • Não use se você quiser fornecer ao usuário uma visão geral de todas as opções disponíveis ou se deseja filtrar por algo ou buscar algo em específico.

Problema

Problema é como poderia agrupar mais de uma instituição para a pesquisa.

Solução

A solução é esse componente com o auxílio da biblioteca de autocomplete. Esse componente une todos os resultados selecionados pelo autocomplete. Passando como parâmetro para a pesquisa.

Selecione vários componentes para a pesquisa, fazendo agrupamentos como parâmetro.

Quando Usar

  • Use quando desejar montar uma hierarquia de acordo com os itens - órgão superior, órgão subordinado, unidade gestora, programa, ação, favorecido.
  • Use quando deseje montar um histótico baseados nos itens - órgão superior, órgão subordinado, unidade gestora, programa, ação, favorecido.
  • Use quando deseje retornar uma agregação baseada na busca feita.
  • Não use for importante visualizar o histórico dos gastos.

Busca Hierárquia

Problema

Para montar uma hierarquia baseada em itens conectados é necessário uma forma de selecionar e montar essa hierarquia proposta.

Solução

Baseado na hierarquia na base de pagamentos que o órgão superior X possui um ou mais orgãos subordinados que por sua vez possui um ou mais unidades gestoras. Uma unidade gestora recebe dinheiro através de um ou mais programas que por sua vez é regido por um ou mais ações que tem como receptor um ou mais favorecido.

Quando Usar

  • Use quando desejar montar uma hierarquia de acordo com os itens - órgão superior, órgão subordinado, unidade gestora, programa, ação, favorecido.
  • Não use quando desejar ver um histórico temporal dos gastos de forma individual de algum item na hierarquia na base de pagamentos.

Visualização dos Gráficos

Problema

O problema é a dificuldade na visualização dos diversos dados, como poderia ver de forma mais simples o conjunto desses dados volumosos.

Solução

A solução foi criar um mapa interativo, onde o usuário pode visualizar todos os gastos/dados de forma simples e rápida. Podendo também limitar o período para a visualização.

Visualize o gráfico, para alterar o período basta clicar e arrastar nos limites do mapa.

Quando Usar

  • Use quando desejar ver os gastos de algum item na hierarquia na base de pagamentos de forma anual, ou mensal.
  • Use quando desejar comparar gastos de forma mais direta entre diferente órgãos, programas ou ações.
  • Não use se desejar ver um montante anual de um orgão, unidade gestora, ação ou programa.

Barra de carregamento

Problema

O problema é a não visualização das páginas e requisições, ou seja, determinada consulta está sendo processada, mas não tem como ver que a mesma ainda está sendo carregada.

Solução

A solução foi colocar uma barra de carregamento que mostra ao usuário do site que determinada tarefa está em execução ou em processamento. Mostrando assim de certa forma um feedback do que está acontecendo ao demorar responder uma requisição.

Visualize a barra enquanto a requisição é completada e retornada.

Quando Usar

  • Use quando desejar informar um retorno ao usuário do que está acontecendo.
  • Use quando uma consulta agregada demorar mais de 3 segundos.
  • Não use toda vez que gerar o resultado de uma consulta, mudança de páginas.

Modal

Problema

O problema quando o usuário faz algo não esperado ou que não é correto no contexto no sistema algo precisa ser feito e o usuário precisa ser informado de sua ação.

Solução

A solução foi apresentar para o usuário no momento que ele faz uma ação inesperada um modal informando uma mensagem com o que é preciso ser feito para completar a ação com sucesso.

Quando Usar

  • Use quando for necessário informar ao usuário, após uma ação indevida, uma mensagem informando o que é preciso ser feito.
  • Não use se desejar mostrar ou descrever como fazer uma ação ao usuário se ele tiver feito a ação de forma correta.

Quais cores são necessárias para que uma boa parte da população que possui algum tipo de deficiência visual ou daltonismo possa entender o que a ferramenta quer passar e não se perder no aspecto visual da ferramenta.

Solução

Foi feito um estudo em que foi observado as principais cores que diferentes níveis de daltonismo enxerga, chegando na paleta de cores em destaque abaixo no exemplo.

Quando Usar

  • Use cada cor em diferentes contrastes como fundo cinza, o verde mais escuro paraa cards, botões, textos, branco para nome do texto dentro dos botões, verde mais claro para cabeçalho, rodapé.

Pagination

Problema

O retorno de uma consulta pode ser grande muitas vezes, sendo assim é necessário uma forma de não deixar uma página extensa e além disso fazer com que o usuário tenha um retorno do que é procurado mais rápido.

Solução

A paginação retorna ao usuário os resultados em parte, podendo ele avançar para outras páginas com os resultados da pesquisa de forma sequencial ou simplesmente ir para a primeira ou última página contendo os resultados da pesquisa.

Quando Usar

  • Use quando o resultado de uma consulta é grande.
  • Use quando desejar destacar alguns resultados dentre muitos outros.
  • Não use se o resultado retornado por uma consulta é sempre pequeno, e o conteúdo retornado é disposto em uma página sem necessidade do usuário fazer muito esforço para ler todo conteúdo.

Tabela

Problema

Quando é preciso dispor as informações de forma lado a lado, mostrar ligações ou agrupamentos é preciso dispor tais informações de uma forma simples e intuitiva.

Solução

Através de tabelas dispomos as informações mais relevantes agrupadas além de acrescentá-las em um só local.

Quando Usar

  • Use quando necessite juntar vários resultados de pesquisas de forma agrupada e lado a lado.
  • Não use se os resultados podem ser melhor visualizados de forma individual e gráfica, ou que não muitas agregações de resultados para serem dispostas em diversos atributos.
